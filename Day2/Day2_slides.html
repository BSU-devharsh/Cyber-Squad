The CIA triad

The three goals of all security

Security professionals judge every system against three goals. Confidentiality means only authorized people can view data, enforced with encryption, access control, and good passwords; a leaked database breaks it. Integrity means only authorized people can change data and that tampering can be detected, enforced with hashing and digital signatures; altering a bank balance breaks it. Availability means authorized people can use the data when they need it, protected with backups and redundancy; a denial-of-service flood breaks it. Every attack you study damages at least one of these pillars, and every defense protects one or more, so the triad is the lens used to reason about risk.

How passwords are attacked

Brute force, dictionaries, and reuse

Attackers rarely guess a password by hand. In a brute-force attack a program tries every possible combination; in a dictionary attack it tries lists of likely words and known leaked passwords first. Speed is the key danger: against stolen password hashes an attacker can test billions of guesses per second on graphics hardware, offline and undetected. Credential stuffing reuses username and password pairs from one breach against many other sites, which is why reusing passwords is so dangerous. Online guessing against a live login is slower and can be blocked by lockouts and rate limits.

Why length beats complexity

Keyspace and entropy

The number of possible passwords, called the keyspace, equals N to the power L, where N is how many characters are allowed and L is the length. Because L is an exponent, adding one character multiplies the total far more than swapping a letter for a symbol does. For example, an eight-character lowercase password has about 209 billion combinations, but a sixteen-character one from the same set is astronomically larger. This is why a long passphrase of several random words is both easier to remember and far stronger than a short complicated password. Entropy is the formal measure of this unpredictability, counted in bits.

Hashing

One-way fingerprints

A cryptographic hash function such as SHA-256 turns any input into a fixed-length fingerprint that cannot be reversed back into the original. Good functions are deterministic (same input, same output), fast to compute, and practically impossible to invert or to find two inputs that collide. Websites store the hash of your password, never the password itself, so that a database leak does not immediately hand attackers your secret. Hashes also verify file integrity: if even one byte changes, the fingerprint changes completely, which is the avalanche effect.

Salting

Defeating precomputed attacks

A salt is random data, unique to each user, added to a password before it is hashed. Without salts, two people with the same password would produce the same hash, and attackers could crack many accounts at once using precomputed tables called rainbow tables. A unique salt forces the attacker to attack each password separately and makes those giant tables useless. Modern systems also use slow, salted password-hashing functions such as bcrypt or Argon2 to make each guess expensive, slowing attackers even further.

Multi-factor authentication and phishing

Adding a second lock

Authentication can use something you know (a password), something you have (a phone or security key), or something you are (a fingerprint). Multi-factor authentication (MFA) requires two or more of these, so a stolen password alone is not enough to log in, which blocks the majority of account takeovers. Phishing attacks try to defeat this by tricking you into handing over both your password and your one-time code, using urgency, fear, and trust. Red flags include a look-alike sender domain, a generic greeting, spelling errors, and a link whose real destination differs from its text.

Multi-factor authentication (MFA)

Something you know + have + are

A single password is a single point of failure. MFA requires two or more independent proofs of identity. The three factors are: something you know (password), something you have (phone TOTP code, hardware key), and something you are (fingerprint, face). Even if an attacker steals your password, they still cannot log in without the second factor. NIST SP 800-63 recommends phishing-resistant MFA (FIDO2/passkeys) for high-value accounts.

Credential stuffing

Why unique passwords matter

Credential stuffing is an automated attack where leaked username-password pairs from one site are tried on other sites. Billions of credentials are available in breach databases. If you reuse the same password on multiple sites, one breach unlocks all of them. The defense is simple: a unique strong password for every account, managed by a password manager. haveibeenpwned.com lets you check whether your email appears in known breaches.
