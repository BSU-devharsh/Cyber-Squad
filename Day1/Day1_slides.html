What is Python, really?

An interpreted, readable language

Python is a high-level, general-purpose programming language created by Guido van Rossum and first released in 1991. High-level means it hides hardware details so you write ideas, not machine instructions. It is interpreted, so it runs your code line by line with no separate compile step, which makes experimenting fast. In security it is the default glue language: a short script can scan a network, parse a log file, automate a repetitive task, or drive a tool, often in a fraction of the lines another language would need. Its huge standard library and readable syntax are why beginners and professionals both reach for it.

TA note: Python is interpreted, so you can demo live. Open a Colab notebook and type print('Hello Cyber Squad!') in front of the class. Students love seeing it run instantly.

Variables and data types

Naming and storing values

A variable is a name bound to a value stored in memory; the equals sign copies the value on the right into the name on the left. Python is dynamically typed, so a name can hold any type and you do not declare the type up front. The core types are str for text in quotes, int for whole numbers, float for decimals, and bool for True or False. Types matter for correctness and security: the string "80" is not equal to the number 80, and treating user input as the wrong type is a common source of bugs and vulnerabilities. Use the type() function to check what you have.

Loops

Automating repetition

A loop runs the same block of code many times so a human does not have to. A for loop walks through a sequence, such as a list of hosts or characters, while a while loop repeats as long as a condition stays true. The range() function generates number sequences for counting. Loops are the heart of automation: scanning a range of ports, trying a list of inputs, or processing every line of a log are all loops. The same idea drives a CodeCombat hero through a maze: repeat a step until a goal is reached.

TA note: Many students confuse for and while. A for loop has a known end (a list or range); a while loop keeps going until a condition flips. Ask: which would you use to ping every host in a list?

Functions

Reusable building blocks

A function is a named, reusable block of code defined with def. It can take inputs called arguments and hand back a result with return. Functions let you write logic once and call it many times, which keeps programs short, readable, and easier to test. Real security tools are assembled from small functions: one to check a single port, another to format a report, called together in a loop over many targets. Breaking a problem into functions is the first step in thinking like an engineer.

What is an operating system?

The software layer between you and the hardware

An operating system (OS) is the software that manages hardware and provides an environment for programs to run. Without an OS, you would write instructions in binary directly to the processor. Windows, macOS, and Linux are the three main OS families. Windows dominates personal computing. macOS runs Apple hardware. Linux runs servers, Android phones, supercomputers, and embedded devices. Security professionals choose Linux because it is open source (you can read every line), highly customizable, and the same OS that production servers run.

Files and the file system

How data is organized on disk

A file is a named container of data stored on disk -- text, code, images, or any bytes. A file system is the structure an OS uses to organize files into folders (directories), track where each file lives on disk, and control who can access it. Think of a file system as a giant filing cabinet: each drawer is a directory, each folder inside is a subdirectory, and each sheet of paper is a file. On Linux the tree starts at the root directory written as a forward slash. Every file has a path -- a sequence of directories from root to the file -- such as /home/student/notes.txt.

TA note: The concept of a path trips students up. Draw it on the board as a tree, root at top, branches going down.

The Linux file system

Everything branches from root

Linux organizes files in a single tree that starts at the root, written as a forward slash. Folders (directories) branch from there, and almost everything, including devices and settings, is represented as a file you can read or write. You move with cd, list with ls, see where you are with pwd, and read a file with cat. Every file carries permissions that say who may read, write, or execute it, which is a core security control. Servers are managed from this command line because it is fast, precise, and easy to script.

TA note: Draw the tree on the board. Start at / then show /home, /etc, /bin. Ask: where would a password file live? They guess /etc.

How a network message travels

Packets, addresses, and routing

When one machine sends data to another, the data is split into small chunks called packets. Each packet carries a header with the source and destination IP addresses, like an envelope with to and from lines, wrapped around a piece of the actual data. Routers read those headers and forward each packet hop by hop toward its destination, where the packets are reassembled in order. The Domain Name System (DNS) first turns a human name such as example.com into the numeric IP address the network needs. In the Cyber Range, all of this happens inside an isolated network, so nothing reaches the public internet.

The OSI and TCP/IP models

OSI modelTCP/IP model
Layers74
Purposea teaching and troubleshooting referencewhat the internet actually runs on
Top layersseparate Application, Presentation, Sessionfolded into one Application layer
Bottom layersseparate Data Link and Physicalcombined into Link / Network Access
Example usename the layer where a problem livesthe model behind real protocols (IP, TCP, HTTP)
Why two models? The OSI model is the seven-layer vocabulary engineers use to describe and debug networks; the TCP/IP model is the simpler four-layer stack the internet is built on. Data is wrapped (encapsulated) layer by layer going down the sender stack and unwrapped going up the receiver stack.
TA note: Use the envelope analogy. An IP packet is like a letter with a to and from address on the outside. Routers are like post offices that read the address and pass it along.

Making decisions

The if statement

if runs code only when a condition is True; else covers the rest.

if len(pw) < 12:
    print("Too short")
else:
    print("Good")
